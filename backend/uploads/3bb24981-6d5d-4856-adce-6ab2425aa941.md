# Unified — Unified Access Control & Enterprise Management Platform

A full-stack enterprise web application that provides a single secure gateway for accessing and managing multiple business software modules: Custom Portal, Asset Lifecycle Management (ALM), Service Desk, and Enterprise Performance Management (EPM).

---

## What It Does

Unified lets organisations manage all their enterprise tools from one place. Instead of juggling multiple logins and systems, users sign in once and get access to the modules their license and role permits. Administrators can control who sees what, enforce security policies, manage licenses, and audit all activity — all from a built-in admin console.

---

## Key Features

### Authentication & Security
- Username/password login with bcrypt hashing
- Multi-Factor Authentication (TOTP) with QR code setup
- Account lockout after repeated failed attempts
- Configurable password policy, session timeout, and IP restrictions
- Secure session management via `express-session`

### Role-Based Access Control (RBAC)
- Hierarchical roles with fine-grained permission sets
- Predefined roles per module (Administrator, Operator, Viewer, Agent, etc.)
- Wildcard `*` permission for system admins
- Role assignment and management via admin UI

### License Management
- Hardware-bound license keys (CPU, hostname, MAC, instance ID fingerprinting)
- Module-specific activation (Custom Portal, ALM, Service Desk, EPM)
- Per-module and overall expiry date tracking
- Local license validation (no server round-trip required after activation)
- System admin users bypass all license restrictions

### Admin Console
| Area | Features |
|---|---|
| User Master | Create, edit, deactivate users; assign roles; reset passwords |
| Role Master | Create roles; configure menu-level permissions |
| Security Settings | Password policy, MFA enforcement, session timeout, account lockout |
| Audit Logs | Full activity log with filtering and cleanup |
| License Settings | Activate licenses, view module expiries, check hardware binding |
| Notification Settings | Push notification and in-app channel configuration |
| Organization Settings | Branding, logo, tagline, colour scheme |
| System Monitoring | System health and usage metrics |

### Modules
- **Custom Portal** — Configurable widget-based landing portal
- **Asset Lifecycle Management (ALM)** — Six dashboards (Overview, Hardware, Software, CIO, CFO, COO) with KPIs, asset tracking, procurement, maintenance, and decommissioning workflows
- **Service Desk** — Incident and request management, SLA tracking, knowledge base, change and problem management
- **Enterprise Performance Management (EPM)** — Employee productivity monitoring, AI scoring, behavioural analytics, reporting

### Other
- In-app notifications with a real-time notification bell
- Dark/light mode support
- Responsive design

## How to Run

### Prerequisites
- Node.js 20+
- PostgreSQL database (or use the Neon serverless connection string)

### 1. Install dependencies

```bash
npm install
```

### 2. Set up environment variables

Create a `.env` file in the project root:

```env
DATABASE_URL=postgresql://user:password@host:5432/dbname
SESSION_SECRET=your-strong-random-secret
NODE_ENV=development
PORT=5000
```

### 3. Push the database schema

```bash
npm run db:push
```

### 4. Start the development server

```bash
npm run dev
```

The app runs at `http://localhost:5000` — the Express backend and Vite frontend are served from the same port.

### Default Credentials

| Username | Password | Role |
|---|---|---|
| `admin` | `P@ssw0rd@123` | System Administrator |

> **Important:** Change the admin password immediately after first login.

## Docker Deployment

A full production setup is included in the `deploy/` folder.

```bash
cd deploy

# One-command setup (includes PostgreSQL, Redis, Nginx)
./setup.sh

# Or manually with Docker Compose
cp .env.example .env        # Edit DATABASE_URL and SESSION_SECRET
docker compose -f docker-compose.full.yml up -d --build
```

Access at `http://localhost` after containers start.

See [`deploy/README.md`](deploy/README.md) for Kubernetes and Docker Swarm instructions.

## License Testing

Use these keys with the test license endpoint to activate modules during development:

| Key | Modules |
|---|---|
| `TEST-ALL-MODULES-2025` | All modules |
| `TEST-PORTAL-ONLY-2025` | Custom Portal |
| `TEST-ALM-ONLY-2025` | Asset Management |
| `TEST-SD-ONLY-2025` | Service Desk |
| `TEST-EPM-ONLY-2025` | EPM |
| `TEST-EXPIRED-2024` | Expired (for expiry testing) |

Set `LICENSE_SERVER_URL` in your environment to your app's base URL + `/api/test/license` to use these keys.
